node指令与nvm

发表于 2017-09-04

Node（从百度开始）

网站 TJ

  • 官网
  • 中文官网
  • 社区
  • node 是从09年开始才有的
  • 深入浅出Node.js (朴灵 著)

Node 版本安装与下载

  • node以前就有两个版本 0.1 node.io(0.5)
  • node 4、6、8(偶数 稳定版本) 5、7、9（奇数 预览版本）
  • npm node package manager (node 包管理工具)
  • nvm node version manager (node 版本管理工具)
  • 推荐看英文版,
  • 最上面有个 DOWNLOADS,点击进入,Windows Installer (.msi)是直接进入安装,没有安装包,Windows Binary (.zip)时下载安装包,来安装
  • 如果要下载所有的版本,在All download options里查看,有历史所有的版本,下载有zip结尾的

nvm安装

  • 安装在c盘,任何英文软件不要有中文路劲,不要有空格,
  • 步骤 :
    1. 先配置nvm安装 路劲
      c:\node\nvm
    2. 再配置node安装路劲
      c:\node\nodejs
      再点下一步,

nvm使用,并切换安装nodejs版本

nvm 切换的版本都是node版本, cmd指令

  • nvm ls 查看有哪些node已经安装了
  • nvm install <版本号> 64/32 安装对应的node,如nvm install 6.9.5,64为系统可以省略写64,
  • nvm uninstall <版本号> 删除包
  • nvm use 版本号 切换包
  • node -v查看现在node的版本
  • 如果是通过下载安装包,要解压放到安装目录中,并改名为下载的版本号如v4.8.3
    安装目录如 ; C:\node\nvm.然后通过版本切换,换自己想要的版本

cmd指令 输入 node 编写代码

  • 如:var a=100 回车,
    var b=200 回车,
    a+b 回车,
    就可以输出结果,但不可以保存,
  • 但是可以在编辑器里建一个js文件写上jsd代码,然后打开这个jsm目录,shift鼠标右键打开cmd窗口,输入node 你的js文件名就可以输出结果,但node没有bom,dom ,window操作,注意写法
  • 浏览器页可以,不用输入node

REPL 可交互环境

    1. 通过cmd 命令可以直接使用node编写代码，但是这种直接编写不能很方便的保存
    1. 通过创建js文件，在js文件中编写js代码 然后通过 node app.js 运行你的代码

node工具 加载本地文件 ,服务器的方式加载 cmd指令(重点)

  • npm install -g http-server
  • 这时候会在你的版本文件夹那里多出4个文件,
    如v6.9.5下多出hs hs.cmd http-server http-server.cmd,还有node_modules文件夹下多出http-server文件夹,
    这是当前版本的如果要切换版本要重新再其他版本下载,
  • http-server 简写 hs
  • 这个工具用来将你当前的文件，以服务器的方式加载,
  • 你需要加载哪个文件夹就在哪个文件夹里shift加右键出cmd窗口,然后输入 http-server就会生成
  • 192.168.67.54:8081类的本地服务器,在浏览器输入192.168.67.54:8081,正个班都可以使用看都你的文件,
  • http-server -port 9000或者hs -port 9000 可以改端口别人就不可访问了,
  • http-server或者hs 重开端口
  • 如果你家的wife和你的手机在同一个局域网下,就可以访问
  • 如192.168.67.54:8081,的端口,这时候你的手机能访问,电脑也能访问,就相当与搭建一个小型服务器,语node环境运行的

node（服务器） 与浏览器不同

    1. 没有dom bom window
    1. this指向也不再指向window，空对象
    1. node可以直接读取本地文件

hexo博客制作

发表于 2017-09-04

Hexo 博客生成器

  • Hexo官网
  • 台湾人开发. 有比较详尽的中文文档.
  • hexo是基于nodejs开发的一个工具.

1.Hexo安装

  1. 使用npm工具全局安装hexo环境 win -r

    执行cmd指令 npm install hexo-cli -g`

  2. 使用 hexo -v 查看hexo的版本 如果有显示版本 则说明安装成功.

  3. 切换到你要创建博客的目录 如:桌面 : cd 到桌面

    执行cmd指令 hexo init myBlog
    myBlog可以使用其他名字

  • 会在当前目录下创建1个myBlog文件夹. 并联网下载一些初始化文件
  • 初始化1个本地仓库 然后从1个仓库中克隆文件到我们的目录下.
  1. 执行cmd指令 hexo server
    开启服务 会有个网址在cmd 里

  • 打开提示的网页地址 就可以看见博客页面了

    ​

2.发表新文章

  • 使用命令来发表新文章
  1. 终止处理操作ctrl +c
  1. 执行cmd指令 hexo new 第一次使用hexo
    就会生成一个新文章, (第一次使用hexo)文章的标题
  • 会在\Desktop\myblog\source_posts\第一次使用hexo 生出一个.md文件
  • 使用md语法编辑 第一次使用hexo .md文件. 该md文件的内容就是发布的博客.
  • 写完后保存关闭第一次使用hexo .md文件
    执行cmd指令 hexo server
    然后刷新一下网站就可以了 重复 2操作生成多篇文章

3. next主题配置

  • 许多第三方为hexo提供了主题, 我们可以搜索并使用.
  • 其中next主题比较受欢迎
  • next主题官网: 点击跳转
  • 点击开始使用 ,下载稳定版 下有 前往 NexT 版本 发布页面。
    翻到 Downloads 下载Source code (zip)
    解压放到 站点目录–博客myBlog themes下 并重命名为next
  • 打开 站点配置 文件就是这个 _config.yml文件
  • 修改其中字段为 theme: next 记得要加空格
  • 执行cmd指令 hexo server 重启服务
  • 刷新网页,就可以了
  • 还有其他的主题 就要修改 主题配置文件 文件根目录下 next文件下有 _config.yml文件,打开找到 Schemes ,#是注释,需要哪个放开哪个,
    修改主题不需要重启服务,直接刷新网页就可以了

    修改为 language: zh-Hans ,有空格
  2. 重启服务, cmd指令,hexo server

5.设置头像,

  1. 修改主题配置文件,找到 avatar, 去掉#

    在avatar后,可以给,网络图片地址,百度想要的图片,点击图片右键复制图片地址,如https://ss1.bdstatic.com/70cFvXSh_Q1YnxGkpoWK1HF6hhy/it/u=2895780816,1616710955&fm=27&gp=0.jpg

    然后把图片放在主题目录下 next\source\images下
    然后修改为 avatar: /images/cover.jpg

    刷新网页,就可以了

6.设置作者

  1. 站点配置文件,_config.yml文件

10.以博客部署到网站去,

  1. 把博客生成一套html代码
    cmd中 hexo g 就生成了一套html代码

    会在站点文件目录,myBlog多一个文件夹 public,
    里面就是生成的一套html代码,
    只需要把他部署到服务器就可以了 github

11.以项目的方式将我们的网站部署到github上去.

第一种

  1. 在github上面创建1个远程的仓库
  2. 初始化你的博客文件夹,使用 git工具
  3. cd 到 博客文件夹,
  4. 初始化一个仓库, git指令
    git init
  5. 创建一个孤立的分支
    git checkout –orphan gh-pages
    然后按提交顺序
    git add .
    git commit -m 我的博客
    git push xxxxxxxxxxxxxx(github创建的仓库) gh-pages
    必须是远程的gh-pages分支,不是master分支,这样才能生成网站地址,
    master分支不生成网站地址
  6. 然后在github上传的那个仓库,点击settings下拉找到网址就可以了

第二种

  1. 创建一个仓库,用 GitHub的用户名.github.io 为仓库名
  2. 然后在博客文件夹
    git init
    git add.
    git commit -m 我的博客
    git push xxxxxxxxxx master
  3. GitHub的用户名.github.io就是网址了

第三种 博客里部署 一劳永逸的做法

  1. 在站点文件下 _config.yml文件,
    找到deploy:

  2. 在cmd指令中装 一个工具
    npm install hexo-deployer-git –save

  3. deploy: 下加多3个属性

  4. 在cmd指令 ,
    生成一个html代码文件夹,网站
    hexo g
    在博客目录下生成一个public文件夹,
    然后
    hexo d
    就会部署到指定的仓库 repo: https://github.com/caishaohui/blog.git的位置

  5. 打开github仓库,找到setting 下拉找到网址,就可以了
    以后改动只要
    hexo d 就行了

gulp使用文档总结

发表于 2017-09-04

gulp

1. gulp简介

  • gulp这个软件是用nodejs写的.
  • gulp是基于流的自动化构建工具
  • 网站开发完成之后,我们要做项目构建,完成之后才可以上线.
  • 项目构建:
    • 代码压缩.html css js
    • 代码混淆
    • 文件合并
    • 等其它自动化工作. sass转换为css.
  • 这就是gulp做的事情.

2. gulp安装

  • 首先需要全局安装gulp,如果之前已经全局安装过,可以省略这步.

    1. 还需要本地安装,在项目目录中.

3. gulp使用

    1. 在项目目录中新建1个gulpfile.js文件.

  • 在这个文件中写上构建代码.

  • 先引入本地的gulp模块

3.1 创建任务

  • gulp是以任务的形式来执行每一项构建化工作.

  • 调用gulp对象的task方法可以创建任务
    参数1: 任务名称
    参数2: 回调函数,执行该任务的时候,要做的事情.

  • 这个时候,我们就创建了1个任务testTask,该任务做的事情就是回调函数的事情.

3.2 执行任务

  • 如何执行这个gulp任务呢?

    1. 打开cmd工具,切换工作路径到项目目录, 是gulpfile.js`文件的目录

  • 在命令行窗口中使用gulp 任务名称就可以执行指定的任务.

  • 这个时候,名叫testTask的任务就会被执行.

4.压缩CSS gulp-cssmin

    1. 先下载 包 npm install gulp-cssmin –save

5.文件监视 watch

  • gulp对象提供了一个watch方法

  • 该方法的作用,监视指定文件的变化,

  • 一旦改动,就执行指定的任务

  1. gulp.task(“watchCss”,function () {
    //监视指定的css文件.可以使用通配符,一旦文件发送变化,就自动执行yscss任务
    gulp.watch(“./src/assets/css/reset.css”,[“yscss”]);

gulp.src(“./js/app.js”)
