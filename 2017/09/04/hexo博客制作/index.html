hexo博客制作

发表于 2017-09-04

Hexo 博客生成器

  • Hexo官网
  • 台湾人开发. 有比较详尽的中文文档.
  • hexo是基于nodejs开发的一个工具.

1.Hexo安装

  1. 使用npm工具全局安装hexo环境 win -r

    执行cmd指令 npm install hexo-cli -g`

  2. 使用 hexo -v 查看hexo的版本 如果有显示版本 则说明安装成功.

  3. 切换到你要创建博客的目录 如:桌面 : cd 到桌面

    执行cmd指令 hexo init myBlog
    myBlog可以使用其他名字

  • 会在当前目录下创建1个myBlog文件夹. 并联网下载一些初始化文件
  • 初始化1个本地仓库 然后从1个仓库中克隆文件到我们的目录下.
  1. 执行cmd指令 hexo server
    开启服务 会有个网址在cmd 里

  • 打开提示的网页地址 就可以看见博客页面了

    ​

2.发表新文章

  • 使用命令来发表新文章
  1. 终止处理操作ctrl +c
  1. 执行cmd指令 hexo new 第一次使用hexo
    就会生成一个新文章, (第一次使用hexo)文章的标题
  • 会在\Desktop\myblog\source_posts\第一次使用hexo 生出一个.md文件
  • 使用md语法编辑 第一次使用hexo .md文件. 该md文件的内容就是发布的博客.
  • 写完后保存关闭第一次使用hexo .md文件
    执行cmd指令 hexo server
    然后刷新一下网站就可以了 重复 2操作生成多篇文章

3. next主题配置

  • 许多第三方为hexo提供了主题, 我们可以搜索并使用.
  • 其中next主题比较受欢迎
  • next主题官网: 点击跳转
  • 点击开始使用 ,下载稳定版 下有 前往 NexT 版本 发布页面。
    翻到 Downloads 下载Source code (zip)
    解压放到 站点目录–博客myBlog themes下 并重命名为next
  • 打开 站点配置 文件就是这个 _config.yml文件
  • 修改其中字段为 theme: next 记得要加空格
  • 执行cmd指令 hexo server 重启服务
  • 刷新网页,就可以了
  • 还有其他的主题 就要修改 主题配置文件 文件根目录下 next文件下有 _config.yml文件,打开找到 Schemes ,#是注释,需要哪个放开哪个,
    修改主题不需要重启服务,直接刷新网页就可以了

Schemes

scheme: Muse

scheme: Mist

scheme: Pisces

4. next设置语言

  1. 找到站点配置文件,_config.yml文件
    修改为 language: zh-Hans ,有空格
  2. 重启服务, cmd指令,hexo server

5.设置头像,

  1. 修改主题配置文件,找到 avatar, 去掉#

    在avatar后,可以给,网络图片地址,百度想要的图片,点击图片右键复制图片地址,如https://ss1.bdstatic.com/70cFvXSh_Q1YnxGkpoWK1HF6hhy/it/u=2895780816,1616710955&fm=27&gp=0.jpg

  2. 站点内部图片,就是本地图片,
    然后把图片放在主题目录下 next\source\images下
    然后修改为 avatar: /images/cover.jpg

    刷新网页,就可以了

6.设置作者

  1. 站点配置文件,_config.yml文件
    title: 蔡少辉的博客 网站的名称
    subtitle: 这里是隔壁老蔡的情感宣泄的地方,欢迎大家来留言 子标题
    description: 真正的隔壁老蔡,你值得拥有,送你一个原谅帽 描述
    author: 蔡少辉 作者
    language: zh-Hans
    timezone: 时区,不管他,会自动设置为自己电脑的时间
  2. 重启服务 cmd中 hexo server ,刷新网页可以了,如果出现乱码,在编辑器改成utf-8编码重写就可以了,

7.设置菜单,默认只有3个,

  1. 找到主题配置文件\next_config.yml文件,menu字段,把#都去掉,刷新网页,就可以了
    menu:
    home: /

    categories: /categories

    about: /about

    archives: /archives
    tags: /tags

    sitemap: /sitemap.xml

    commonweal: /404.html

8.新增菜单

  1. 在menu下按照格式写就行了,如加一个photo
    menu:

  2. 还需要在\next\languages\zh-Hans.yml中,后加 photo: 照片,就可以了
    menu:
    home: 首页
    archives: 归档
    categories: 分类
    tags: 标签
    about: 关于
    search: 搜索
    schedule: 日程表
    sitemap: 站点地图
    commonweal: 公益404
    photo: 照片

  3. 设置新增菜单的字体图标
    在主题目录下\next_config.yml文件,加上 photo: image
    怎么知道字体图标名?百度font awesome 或在这 http://fontawesome.dashgame.com/找
    然后把图标名image等放到后面就可以了
    menu_icons:
    enable: true //false所有的图标不显示

    KeyMapsToMenuItemKey: NameOfTheIconFromFontAwesome

    home: home
    about: user
    categories: th
    schedule: calendar
    tags: tags
    archives: archive
    sitemap: sitemap
    commonweal: heartbeat
    photo: image

9.点击菜单跳转到一个页面

  1. 例如点击刚创建的 photo照片
  2. 在cmd指令 hexo new page photo
    会在\Desktop\myblog\source\photo\ 生出一个.md文件
    使用md语法编辑 .md文件. 该md文件的内容就是发布的博客.
  3. 写完后保存关闭.md文件
    执行cmd指令 hexo server
    然后刷新一下网站就可以了 重复 2操作生成多篇文章

10.以博客部署到网站去,

  1. 把博客生成一套html代码
    cmd中 hexo g 就生成了一套html代码

    会在站点文件目录,myBlog多一个文件夹 public,
    里面就是生成的一套html代码,
    只需要把他部署到服务器就可以了 github

11.以项目的方式将我们的网站部署到github上去.

第一种

  1. 在github上面创建1个远程的仓库
  2. 初始化你的博客文件夹,使用 git工具
  3. cd 到 博客文件夹,
  4. 初始化一个仓库, git指令
    git init
  5. 创建一个孤立的分支
    git checkout –orphan gh-pages
    然后按提交顺序
    git add .
    git commit -m 我的博客
    git push xxxxxxxxxxxxxx(github创建的仓库) gh-pages
    必须是远程的gh-pages分支,不是master分支,这样才能生成网站地址,
    master分支不生成网站地址
  6. 然后在github上传的那个仓库,点击settings下拉找到网址就可以了

第二种

  1. 创建一个仓库,用 GitHub的用户名.github.io 为仓库名
  2. 然后在博客文件夹
    git init
    git add.
    git commit -m 我的博客
    git push xxxxxxxxxx master
  3. GitHub的用户名.github.io就是网址了

第三种 博客里部署 一劳永逸的做法

  1. 在站点文件下 _config.yml文件,
    找到deploy:

  2. 在cmd指令中装 一个工具
    npm install hexo-deployer-git –save

  3. deploy: 下加多3个属性

  4. 在cmd指令 ,
    生成一个html代码文件夹,网站
    hexo g
    在博客目录下生成一个public文件夹,
    然后
    hexo d
    就会部署到指定的仓库 repo: https://github.com/caishaohui/blog.git的位置

  5. 打开github仓库,找到setting 下拉找到网址,就可以了
    以后改动只要
    hexo d 就行了
